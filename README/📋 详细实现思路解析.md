## 📋 详细实现思路解析

### **第一层：项目整体架构**

这个 Web IDE 采用**前后端分离 + WebSocket 实时通信** 的架构：

- **前端（React + Vite）**：提供代码编辑、文件管理、实时协作界面
- **后端（Node.js + Express）**：处理认证、代码执行、多房间隔离、实时广播
- **通信方式**：HTTP（认证） + WebSocket（实时编辑同步 + 代码执行）

------

### **第二层：核心功能流程（4个阶段）**

#### **1️⃣ 登录认证阶段**

```jsx
用户输入昵称+房间号 
  ↓
POST /api/join (Axios)
  ↓
后端签发 JWT Token (jsonwebtoken 库)
  ↓
localStorage 存储 Token
  ↓
触发建立 WebSocket 连接
```

**涉及知识点**：

- JWT 身份验证机制
- localStorage 本地存储
- async/await 异步流程

#### **2️⃣ WebSocket 长连接认证**

```jsx
前端发起连接：io('http://localhost:3000', { auth: { token } })
  ↓
后端中间件 io.use() 拦截验证
  ↓
JWT 验证通过 → socket.user 挂载用户信息
  ↓
socket.join(roomId) → 加入房间
```

**涉及知识点**：

- WebSocket 握手机制
- Socket.io 房间隔离
- 中间件模式（权限验证）

#### **3️⃣ 实时协作编辑**

```jsx
编辑器输入 (onChange 事件)
  ↓
debounce 防抖 (防止频繁发送)
  ↓
isRemoteUpdate 标识 (避免死循环)
  ↓
socket.emit('codeChange', code) 发送给后端
  ↓
后端 socket.to(roomId).emit() 房间内广播
  ↓
其他用户接收到代码更新
  ↓
Editor.setValue() 更新编辑器显示
```

**核心防护机制**：

```jsx
// 防止无限循环：
if (isRemoteUpdate.current) {
  isRemoteUpdate.current = false;  // 打开锁
  return;  // 拦截发送
}
// 本地输入才发送
setCode(value);  // 触发父组件 handleCodeChange
```

#### **4️⃣ 代码执行阶段**

```jsx
点击运行按钮
  ↓
editorRef.current.getValue() 获取代码
  ↓
socket.emit('executeCode', code) 发送到后端
  ↓
后端处理：
  1. fs.promises.writeFile() → 代码落盘
  2. spawn('node', [filePath]) → 创建子进程
  3. child.stdout.on('data') → 监听输出
  4. child.stderr.on('data') → 监听错误
  ↓
socket.emit('terminalOutput') 实时推送结果
  ↓
前端 Terminal 组件接收并展示
```

#### **防抖机制（Debounce）**

```jsx
// 用 lodash 防抖：每 500ms 最多发一次请求
const debouncedSendCode = debounce(() => {
  socket.emit('codeChange', code);
}, 500);
```

#### **后端子进程执行**

```jsx
const child = spawn('node', [filePath], { timeout: 5000 });
// timeout 5秒防止死循环导致服务器卡死
```

#### **房间隔离机制**

```jsx
// 不用全网广播，而是房间广播
socket.to(roomId).emit('codeChange', newCode);
// 这样不同房间的用户代码互不影响
```
