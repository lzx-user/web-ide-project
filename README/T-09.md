### 1. 按照 JWT 规范，生成 JWT (✅ 思路正确)

- **补充细节**：生成 JWT 这一步，通常**不是**在 WebSocket 建立连接时做的，而是在**传统的 Express HTTP  接口**中完成的。
- **流程**：用户访问你的 Web IDE（可能需要登录，或者前端调用一个类似 `/api/init-session` 的接口），后端生成包含用户信息的 JWT（例如载荷里带有 `userId` 和 `username`），并通过 HTTP 响应返回给前端。

### 2. 使用 `auth` 发送凭据 (✅ 思路完全正确)

- **补充细节**：前端拿到 JWT 后，在初始化 Socket.IO 客户端时，将其放入 `auth` 对象中带给后端。

- **伪代码示意**：

  JavaScript

  ```jsx
  // 前端代码
  const socket = io("http://localhost:3000", {
    auth: {
      token: "eyJhbGciOiJIUzI1NiIs..." // 这里放你的 JWT
    },
    query: {
      roomId: "1024" // 顺便把 URL 里的 roomId 也传给后端
    }
  });
  ```

### 3. 使用 `verifyToken` 验证中间件 (✅ 思路正确，需注意语法)

- **纠正与细节**：你可能受 Express 框架影响较深。在 Socket.IO 中，中间件的挂载方式是 `io.use()`，并且参数对象略有不同。你需要从握手信息（`handshake`）中提取 token。

- **伪代码示意**：

  JavaScript

  ```jsx
  // 后端代码
  io.use((socket, next) => {
    // 1. 提取 token
    const token = socket.handshake.auth.token;
  
    // 2. 验证 JWT
    jwt.verify(token, '你的密钥', (err, decoded) => {
      if (err) return next(new Error("Authentication error")); // 拒绝连接
  
      // 3. 验证通过，把用户信息挂载到 socket 对象上，方便后续步骤使用
      socket.user = decoded; 
      next(); // 放行，允许建立连接
    });
  });
  ```

### 4. 通过 `io.join().emit()` 发送特定房间 (❌ 语法有误，逻辑需微调)

这里是你唯一需要纠正代码语法的地方：**`io` 对象没有 `join` 方法，只有 `socket` 对象有。**

- `io` 代表整个服务器（总机）。
- `socket` 代表当前连进来的这个具体的客户端（分机）。

**正确的逻辑是：**

1. **加入房间**：让当前这个连进来的**用户 (`socket`)** 加入某个房间。
2. **广播消息**：通过**服务器 (`io`)** 或当前用户 (`socket`) 向特定房间广播。

至于你提到“用 `userId` 发送特定房间”，对于 Web IDE 这个场景，如果你希望**多个人在同一个房间里协同写代码**，你应该使用 `roomId` 作为房间号，而不是 `userId`（用 userId 建房通常是用来做一对一私聊的）。

- **正确伪代码示意**：

  ```jsx
  // 建立连接后
  io.on("connection", (socket) => {
    // 获取前端传过来的房间号 (比如从 query 中获取)
    const roomId = socket.handshake.query.roomId;
  
    // 1. 【关键纠正】让这个具体的 socket 加入房间，而不是 io.join
    socket.join(roomId);
    console.log(`用户 ${socket.user.username} 加入了房间 ${roomId}`);
  
    // 2. 监听打字事件
    socket.on("code-change", (codeString) => {
      // 3. 【关键操作】只向这个房间里的其他用户广播代码变更
      // socket.to(roomId).emit(...) 表示发给房间里除了自己之外的所有人
      socket.to(roomId).emit("sync-code", codeString); 
    });
  });
  ```







### 🗺️ T-09 房间隔离与鉴权：标准开发顺序路线图

开发这种功能，最好的策略是**“先铺路，再通车”**。我们先写后端的 HTTP 接口（发房卡），再写前端的界面（领房卡），最后搞 WebSocket 长连接（验房卡进房间）。

#### 🏗️ 阶段一：发放通行证（常规 HTTP 阶段）

在这个阶段，我们先不碰 WebSocket，只做一件事：让用户有个地方能输入名字和房间号，并换取一张加密的“房卡”（JWT Token）。

- **大步骤 1：开发后端发卡接口 (Node.js)**
  - *小步骤 1.1*：在后端项目中安装 `jsonwebtoken` 依赖。
  - *小步骤 1.2*：在 Express 中新增一个普通的 POST 路由（例如 `/api/join`）。
  - *小步骤 1.3*：在这个路由里，接收前端传来的“昵称”和“房间号”。
  - *小步骤 1.4*：把昵称和房间号打包，用密钥签发成一个 Token 字符串，通过接口返回给前端。
- **大步骤 2：开发前端大厅界面 (React)**
  - *小步骤 2.1*：改造你的主要组件（比如 `App.jsx`），加一个状态（比如 `isJoined`），用来控制页面是显示“登录大厅”还是“IDE 编辑器”。
  - *小步骤 2.2*：画一个简单的“大厅”界面，包含两个输入框（昵称、房间号）和一个“进入房间”按钮。
  - *小步骤 2.3*：点击按钮时，向后端的 `/api/join` 发送请求。
  - *小步骤 2.4*：拿到后端返回的 Token 后，把它存进浏览器的 `localStorage` 里。
  - *小步骤 2.5*：**关键动作**：用 JS 把浏览器的 URL 修改成带有房间号的格式（例如 `?roomId=1024`），然后把状态切换为“显示 IDE”。

*(🏆 阶段一里程碑：你能看到页面从大厅变成了编辑器，并且浏览器的 LocalStorage 里多了一串 Token。)*

------

#### 🚀 阶段二：升级为安全的长连接（WebSocket 握手阶段）

现在用户已经拿着房卡站在房间门口了，我们需要让他们带着房卡去连接 WebSocket，并在门口设立保安。

- **大步骤 3：前端携带凭据发起连接 (React / Socket.js)**
  - *小步骤 3.1*：在前端初始化 `io()` 连接的代码处，先从 `localStorage` 读取 Token，从 URL 地址栏读取房间号。
  - *小步骤 3.2*：在 `io()` 的配置参数中，把 Token 塞进 `auth` 对象，把房间号塞进 `query` 对象，向后端发起连接。
- **大步骤 4：后端设立保安检查 (Node.js / Socket.io)**
  - *小步骤 4.1*：在后端的 Socket.IO 配置处，加上 `io.use(...)` 拦截中间件。
  - *小步骤 4.2*：从中间件的参数中提取出前端传过来的 Token。
  - *小步骤 4.3*：调用 JWT 库验证这个 Token 的真伪。如果是假的或过期的，直接拒绝连接（抛出 Error）。如果是真的，放行进入下一步。

*(🏆 阶段二里程碑：合法的用户能够成功连上 WebSocket，如果你在前端传个假 Token，后端会拒绝连接并在前端控制台报错。)*

------

#### 🏠 阶段三：物理隔离与定向广播（WebSocket 房间阶段）

保安验证通过了，最后一步就是把用户领进指定的房间，并确保他们说话只有房间里的人能听见。

- **大步骤 5：后端分配房间与广播改造 (Node.js / Socket.io)**
  - *小步骤 5.1*：在后端的 `io.on('connection')`（成功连接的回调）中，提取出握手时传进来的房间号。
  - *小步骤 5.2*：执行 `socket.join(房间号)`，将该用户拉入独立房间。
  - *小步骤 5.3*：找到你之前写的监听 `code-change`（打字同步）的代码。把原本发给所有人的全局广播，替换为只发给该房间的定向广播（使用 `socket.to(房间号).emit`）。

*(🏆 最终里程碑：开两个浏览器进 1024 房间，打字能互相看到。再开一个浏览器进 2048 房间，它里面的打字对 1024 房间毫无影响。T-09 任务完美通关！)*
