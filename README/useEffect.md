`useEffect` 可以说是 React 里最核心、也最容易让人抓狂的一个 Hook。我们不背官方那些晦涩的定义，就拿你刚才写的**“打电话（Socket 连接）”**的代码来举例。

### 1. 为什么叫 “副作用” (Effect)？

React 的主业非常专一，它就是一个**“画图匠”**。它的核心任务只有一个：**根据数据，把 HTML 页面画出来（渲染 UI）。**

但在实际开发中，我们不仅要画页面，还要干很多“杂活”，比如：

- 找后端要数据（发 Ajax 请求）
- 建立 WebSocket 长连接（打电话）
- 设置一个定时器（`setTimeout`）
- 手动修改一下网页的标题（`document.title`）

这些“偏离了画图主业的杂活”，在 React 里统统被称为**“副作用 (Side Effect)”**。而 `useEffect` 就是 React 专门给你提供的一个**“杂活包工头”**。

### 2. 如果刚才不加 `useEffect` 会发生什么灾难？

假设你刚才的代码不写 `useEffect`，直接把拨号的代码写在组件里：

JavaScript

```jsx
function App() {
  // 灾难代码：没有 useEffect
  const socket = io('http://localhost:3000'); // 直接打电话

  return ( <div>页面内容</div> )
}
```

**可怕的后果：** React 有个特点，只要页面上的数据（state）一发生变化，它就会**重新执行一遍 `App()` 函数**来重新画页面。 如果将来你在页面上的代码编辑器里敲键盘，每敲一个字母，React 就会重新执行一次 `App()`。这意味着：**你每敲一个字母，前端就会立刻向后端拨打一次新电话！** 你敲了一百个字母，后端就会瞬间收到一百个连接请求，你的电脑和后端服务器会直接崩溃死机。

### 3. 加了 `useEffect` 之后，它帮你做了什么？

你刚才写的正确代码是这样的：

JavaScript

```jsx
useEffect(() => {
  const socket = io('http://localhost:3000'); // 1. 干杂活：打电话

  return () => {
    socket.disconnect(); // 3. 擦屁股：挂电话
  };
}, []); // 2. 核心控制：[] 表示只在最开始执行一次
```

这个包工头帮你做了三件极其重要的事情：

- **第一件事（干活）：** 它保证了“打电话”这个动作，会在 React 把页面**画完之后**才去执行，绝对不耽误页面显示的加载速度。
- **第二件事（控制次数，看那个 `[]`）：** 最后的那个空数组 `[]` 是灵魂。它等于在告诉包工头：“这个电话，**只有在页面第一次打开的时候打一次就行了**”。以后不管页面怎么刷新、你在编辑器里怎么敲字，都绝对不要再重复拨号了。
- **第三件事（擦屁股，看那个 `return`）：** 当用户关掉这个网页，或者跳转到别的页面时，组件会被销毁。`return` 里面的代码就是告诉包工头：“主人要走啦，记得把电话挂掉（`disconnect`），别浪费电话费（内存泄露）！”

------

**一句话总结：** 当你需要在 React 里做一些**“和单纯画页面无关的杂活”**，并且你需要**控制这个杂活“什么时候干、干几次、什么时候收尾”**时，就必须用 `useEffect`。
