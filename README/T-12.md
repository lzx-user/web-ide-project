实现 T-12（前后端联调并渲染终端面板）的核心在于**事件驱动的数据流转**。以下是具体的实现思路、操作步骤以及相关文档，全程不涉及具体代码。

### 💡 实现思路

这个任务的本质是一个典型的“**用户交互 -> 发起请求 -> 数据更新 -> 视图重绘**”的闭环过程。

1. **视图层 (View) 的触发：** 用户在浏览器界面点击“运行”按钮，产生一个交互事件。
2. **逻辑层 (Logic) 的处理：** 拦截这个点击事件，提取出用户当前编辑的代码内容。
3. **网络层 (Network) 的通信：** 充当信使，将代码数据通过 HTTP 请求发送给后端服务器。同时进入“等待”状态。
4. **状态层 (State) 的更新：** 后端执行完毕返回结果后，前端接收到响应数据，并将这个“执行结果”保存到 React 的状态机中。
5. **视图层 (View) 的重绘：** React 监听到状态发生了改变，自动驱动底部的终端面板组件进行重新渲染，将真实的结果展示给用户。

------

### 🛠️ 拆解步骤

为了顺利完成这个闭环，你需要按照以下顺序逐步实施：

**第一步：准备 React 状态 (State)**

- 在你的组件中声明必要的状态变量。你至少需要一个状态来存储**终端面板需要显示的输出内容**。
- 建议额外增加一个**加载状态 (Loading)**，用于在请求期间禁用“运行”按钮或显示 Loading 动画，防止用户重复点击。

**第二步：绑定事件监听**

- 找到网页上的“运行”按钮组件。
- 为它绑定一个点击事件的处理函数。在这个函数中，你将串联起后续的所有操作。

**第三步：构造并发送 Axios 请求**

- 在点击事件的处理函数中，获取用户编辑器中的当前代码。
- 使用 Axios 发起一个 HTTP 请求（通常是 POST 请求，因为代码内容可能较长或包含特殊字符）。
- 将代码内容作为请求体（Payload）发送给后端指定的 API 接口。

**第四步：处理异步响应与异常**

- 请求发送后，前端需要等待后端的响应。
- **成功情况：** 如果请求成功且后端返回了执行结果，提取出响应体中的输出字符串数据。
- **失败情况：** 必须捕获可能发生的错误（如网络断开、后端报错等），并提取出错误信息。

**第五步：更新状态驱动渲染**

- 无论是成功获取的输出结果，还是捕获到的错误信息，都将它们通过状态更新函数（如 `setState`）写入到第一步准备好的“终端输出内容”状态中。
- 取消“加载状态”。
- 底部的终端面板组件由于绑定了该状态，会自动将最新的真实结果展示在屏幕上。
