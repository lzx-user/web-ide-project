### 怎么让我带你熟悉项目

你直接按这个格式来问我就行，我会给你**面试官视角的问题 + 你应该怎么回答**：

```
我想准备项目面试，请以面试官的角度问我关于
[某个具体模块] 的问题，然后给我标准答案，
让我知道该怎么说。
```

比如：

- "请以面试官角度问我 Yjs 协同编辑怎么实现的"
- "请问我为什么用 Socket.io 而不是原生 WebSocket"
- "请问我 JWT 鉴权的流程"
- "请问我单端口架构怎么做到的"

我会先**扮演面试官问你**，你先尝试回答，然后我给你补充你没说到的点，帮你把答案打磨成面试级别的表达。

### 你项目里高频会被问到的点（提前预告）

面试官最喜欢问这个项目的几个地方：

**技术选型类**：为什么用 Yjs 不用 OT 算法？为什么 Socket.io 和 Yjs 用同一个端口？

**架构类**：控制面和数据面分离是什么意思？upgrade 事件路由怎么工作的？

**React 类**：为什么用 useRef 存 Monaco 实例而不是 useState？fileCacheMap 为什么要用 Map 缓存模型？

**安全类**：路径越界攻击是什么？你怎么防御的？沙箱模块拦截怎么实现的？

**工程类**：Render 冷启动问题怎么处理的？IndexedDB 离线持久化有什么用？
