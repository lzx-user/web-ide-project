### 💡 资深工程师的“破题”思路是怎么建立的？

当你下次遇到一个毫无头绪的需求时，不要去想“我要写什么代码”，而是去画**输入和输出**。

以我们这次的“树形目录”为例：

1. **确定输入：** 后端硬盘里的真实文件夹（里面可能有文件夹，文件夹里还有文件夹...层级未知）。
2. **确定输出：** 前端 Sidebar 组件需要的一个带有 `children` 属性的深层嵌套 JSON 数组。
3. **寻找中间件（API 工具）：** Node.js 提供了什么 API？查文档发现 `fs.readdirSync` 可以读取目录里的东西。
4. **发现核心矛盾：** `fs.readdirSync` 只能读**当前这一层**，如果遇到子文件夹，它不会自己钻进去。
5. **建立技术联想：** 在计算机科学里，处理“未知层级、自己内部包含自己”的结构，标准解法是什么？**递归（Recursion）**。

一旦你想到了“递归”这两个字，思路就彻底打通了：写一个函数，读当前目录。遇到文件就保存；遇到文件夹，就**让这个函数调用它自己**，钻进去继续读。
