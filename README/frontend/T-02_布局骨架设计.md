# T-02：组件化拆分与 Props 通信

## 1、开发思路

### **第一步：建立“总调度室”（App.jsx 状态管理）**

在拆分零件之前，我们需要在 `App.jsx` 里定义好所有的“大脑数据”（状态）。因为在 React 中，数据是**单向流动**的，只有总部握着数据，才能发给下面的零件。

#### 1.1 为什么要定义这些状态？

为了让 IDE 动起来，我们需要记住四个核心信息：

1. **选了哪个文件**：`activeFile`。
2. **写了什么代码**：`currentCode`。
3. **是否正在运行**：`isRunning`。
4. **历史运行日志**：`terminalLogs`。

#### 1.2 编写总部调度代码

**核心知识点（敲黑板）：** 原本所有的 HTML 都在 `App.jsx` 里，现在我们把它们换成“自定义标签”（即子组件）。通过属性（Props）的形式，把数据和修改数据的方法像寄快递一样传下去。

```react
// App.jsx
function App() {
  // 总部握着的四个核心对讲机（状态）
  const [activeFile, setActiveFile] = useState('index.js');
  const [currentCode, setCurrentCode] = useState('...');
  const [isRunning, setIsRunning] = useState(false);
  const [terminalLogs, setTerminalLogs] = useState([]);

  return (
    <div className="h-screen w-screen flex bg-gray-900 ...">
      {/* 传给侧边栏：让它知道谁该高亮，以及怎么切换文件 */}
      <Sidebar activeFile={activeFile} setActiveFile={setActiveFile} />

      <div className="flex-1 flex flex-col">
        {/* 传给顶部：让它显示文件名，并绑定运行按钮的点击事件 */}
        <Header activeFile={activeFile} onRunCode={handleRunCode} isRunning={isRunning} />

        {/* 传给编辑器：把代码塞进去，并接收打字时的变化 */}
        <CodeEditor code={currentCode} setCode={setCurrentCode} />

        {/* 传给终端：把所有的日志数组发给它去展示 */}
        <Terminal logs={terminalLogs} />
      </div>
    </div>
  );
}
```

### **第二步：零件执行与“对讲机”回传（子组件实现）**

每个部门（组件）只需要管好自己那一块。它们通过小括号里的参数（Props）接收总部指令。

#### 2.1 编写侧边栏（Sidebar.jsx）

**为什么要写 `setActiveFile(file.name)`？** 因为子组件没权利直接改总部的数据，它必须通过总部给的这个“对讲机”大喊一声，总部听到后才会更新状态，从而让页面变色。

```react
// Sidebar.jsx
export default function Sidebar({ activeFile, setActiveFile }) {
  return (
    <div className="...">
      {fileList.map((file) => (
        <p
          key={file.id}
          // 点击时，通过 Props 回传选中的名字给总部
          onClick={() => setActiveFile(file.name)}
          // 逻辑判断：如果当前名字等于 activeFile，就变蓝
          className={`... ${activeFile === file.name ? 'bg-blue-600' : 'hover:bg-gray-700'}`}
        >
          {file.icon} {file.name}
        </p>
      ))}
    </div>
  );
}
```

### **使用 `onClick` (瞬间触发)**

- **原因**：切换文件是一个**点状动作**。
- **逻辑**：你只有在鼠标按下那一刻，才需要告诉总部“我要换文件了”。平时鼠标在上面晃来晃去，数据是不变的。
- **比喻**：就像按电梯按钮，你按下去那一刻电梯才知道你要去几楼，按完就结束了。

#### 2.2 编写编辑器（CodeEditor.jsx）

**为什么要写 `setCode(value ?? '')`？**

 Monaco Editor 在清空内容时可能返回 `null`，这会导致程序崩溃。使用 `?? ''` 能保证代码永远是一个字符串，这就是外企要求的**代码健壮性**。

```react
import Editor from '@monaco-editor/react';

// 中间战区
// 职责： 封装 Monaco Editor，处理代码输入。
// 内部逻辑： 它未来需要把写好的代码发送给后端。

export default function CodeEditor({ code, setCode }) {
  return (
    <div className="flex-1">
      <Editor
        height="100%"
        language="javascript"  // 开启 JS 语法高亮，让代码变彩色。
        theme="vs-dark"
        value={code}
        onChange={(value) => setCode(value ?? '')}
        options={{
          fontSize: 16,
          minimap: { enabled: false },   // 关掉右侧小地图，小屏幕下节省空间。
          wordWrap: 'on',   // 自动折行，代码太长时不用拉横向滚动条。
          padding: { top: 16 }
        }}
      />
    </div>
  );
}

```

**使用 `onChange` (持续监听)**

- **原因**：写代码是一个**连续的过程**。
- **逻辑**：你每敲一个字母，编辑器的内容就变了。我们需要在每一毫秒都捕捉到这种“变化（Change）”，并实时存入总部的 `currentCode`。
- **比喻**：就像你在录音，麦克风必须一直开着，捕捉你每一个细微的声音变化。

### **第三步：状态反馈与界面交互**

总部根据实时状态决定给子组件展示什么，形成一个闭环。

#### 3.1 顶部按钮的禁用逻辑（Header.jsx）

**为什么要传 `isRunning`？** 防止用户在代码运行还没结束时疯狂点击，导致服务器压力过大或前端卡死。

```react
// 顶部战区
// 职责： 专注展示标题、当前文件名和“运行”按钮。
// 需要接收的数据 (Props)： 只需要知道当前选中的文件是谁（用来展示名字）。


// 1. 定义组件，并接收总部传来的“快递包裹”
export default function Header({ activeFile, onRunCode, isRunning }) {
  return (
    <div className="h-12 bg-gray-800 border-b border-gray-700 flex items-center justify-between px-6">
      <span className="font-semibold text-gray-400 flex items-center gap-2">
        Web IDE <span className="text-gray-600">/</span>
        <span className="text-gray-200">{activeFile}</span>
      </span>

      // 2. 渲染按钮
      {/* 绑定执行事件，运行时禁用按钮防止重复点击 */}
      <button
        // 3. 点击事件：用户一点，立刻按响总部给的“对讲机” (onRunCode)
        onClick={onRunCode}
        // 4. 禁用逻辑：如果总部说正在运行 (isRunning)，按钮就锁死，不让点
        disabled={isRunning}
        // 5. 动态变色：正在跑就变灰色 (bg-gray-600)，没跑就是绿色 (bg-green-600)
        className={`px-5 py-1.5 rounded text-sm font-bold shadow-lg transition-all ${isRunning
          ? 'bg-gray-600 cursor-not-allowed text-gray-300'
          : 'bg-green-600 hover:bg-green-500 active:scale-95 text-white'
          }`}
      >
        {/* 6. 动态文字：根据状态显示是“运行中”还是“运行”按钮 */}
        {isRunning ? '⏳ 运行中...' : '▶ 运行 (Run)'}
      </button>
    </div>
  );
}
```

#### 3.2 终端日志的分类展示（Terminal.jsx）

**为什么要判断 `log.type`？** 为了让用户一眼分清什么是普通信息，什么是严重的错误。

```react
// Terminal.jsx
export default function Terminal({ logs }) {
  return (
    <div className="...">
      {logs.map(log => (
        <p 
          key={log.id} 
          // 根据后端传回的 type 动态切换颜色
          className={`${log.type === 'error' ? 'text-red-400' : 'text-green-400'}`}
        >
          {log.text}
        </p>
      ))}
    </div>
  );
}
```
