# Web IDE 开发文档 - T-04: 组件化拆分与状态交互

## 🎯 核心目标

消除单文件“意大利面条式代码”，将庞大的 `App.jsx` 拆分为职责单一的四个子组件。并引入数据驱动思想和 React `useState`，实现左侧文件树与顶部导航栏的状态联动。

## 🛠️ 开发思路

### 1. 组件拆分架构 (UI 解耦)

在 `src/components/` 目录下建立四个独立的功能模块：

- `Sidebar.jsx`：负责渲染侧边栏文件列表逻辑。
- `Header.jsx`：负责展示当前文件名和相关操作按钮。
- `CodeEditor.jsx`：专注包裹和配置 Monaco Editor。
- `Terminal.jsx`：负责独立渲染终端日志系统。

### 2. 状态提升 (State Management)

遵循 React 单向数据流原则，将核心状态 `activeFile` (当前选中的文件名) 存放在最高的父组件 `App.jsx` 中，然后通过 `Props` 像瀑布一样向下传递给需要的子组件。

### 3. 数据驱动渲染

摒弃写死的 HTML 结构，定义 `fileList` 数组，采用 `.map()` 遍历渲染列表，并使用三元运算符 `condition ? a : b` 动态切换选中与非选中的 CSS 样式。

### 4. 终极架构代码呈现

**总指挥部 (App.jsx)：**

```react
import { useState } from 'react'; // 引入 React 的状态魔法
import Editor from '@monaco-editor/react'; // 引入 Monaco Editor 组件
import Sidebar from './components/Sidebar'; // 引入侧边栏组件
import Header from './components/Header';
import CodeEditor from './components/CodeEditor';
import Terminal from './components/Terminal';

function App() {
  // 🌟 核心魔法：记录当前选中的是哪个文件，默认选中第一个
  const [activeFile, setActiveFile] = useState('index.js');
  return (
    // 最外层容器：撑满全屏 (h-screen w-screen)，采用 Flex 横向布局
    <div className="h-screen w-screen flex bg-gray-900 text-white font-sans">
      {/* 把状态和修改状态的方法，当做参数 (Props) 传给侧边栏 */}
      <Sidebar activeFile={activeFile} setActiveFile={setActiveFile} />

      {/* 🟢 T-02 任务核心 2：右侧主工作区 (占据剩余全部宽度) */}
      <div className="flex-1 flex flex-col">

        {/* 顶部导航只需要知道当前名字，不需要修改它 */}
        <Header activeFile={activeFile} />

        {/* 中间的编辑器 */}
        <CodeEditor />

        {/* 底部终端 */}
        <Terminal />
      </div>
    </div>
  );
}

export default App;
```

**侧边栏子组件 (Sidebar.jsx) 核心交互：**

```react
import React from 'react';

// 左侧区域
// 职责： 专注渲染文件树列表。
// 需要接收的数据 (Props)： 需要知道当前选中的文件是谁（用来高亮），还需要一个能修改当前选中文件的方法（点击时触发）。

const Sidebar = ({ activeFile, setActiveFile }) => {
  // 左侧文件树
  const fileList = [
    { id: 1, name: 'index.js', type: 'file', icon: '📄' },
    { id: 2, name: 'package.json', type: 'file', icon: '📄' },
    { id: 3, name: 'style.css', type: 'file', icon: '📄' }
  ];

  return (
    <div className="w-[250px] bg-gray-800 border-r border-gray-700 flex flex-col">
      <div className="p-4 border-b border-gray-700 font-bold tracking-wider text-gray-300">
        📂 资源管理器
      </div>
      <div className="p-4 text-sm text-gray-400">
        {fileList.map((file) => (
          <p
            key={file.id}
            onClick={() => setActiveFile(file.name)} // 🎯 点击时，改变当前激活的文件名
            className={`cursor-pointer py-1.5 px-2 rounded transition-colors mb-1 ${activeFile === file.name
              ? 'bg-blue-600 text-white' // 如果被选中了，变成蓝色底
              : 'hover:bg-gray-700 hover:text-white' // 如果没被选中，保留你的 hover 效果
              }`}
          >
            {file.icon} {file.name}
          </p>
        ))}
      </div>
    </div>
  );
};

export default Sidebar;

```

**终端子组件(Terminal.jsx)**

```react
import React from 'react';

// 底部战区
// 职责： 专注显示系统日志和代码运行结果。
// 内部逻辑： 它自己管理一个 terminalLogs 的日志数组。

const Terminal = () => {
  // 底部终端
  const terminalLogs = [
    { id: 1, type: 'info', text: '# 终端控制台已启动，等待执行指令...' },
    { id: 2, type: 'success', text: '> [19:50:26] 编译成功，无语法错误。' },
    { id: 3, type: 'error', text: '> [19:50:27] Error: 这是一个测试报错信息！(红色警告)' }
  ];

  return (
    <div className="h-64 bg-[#0d1117] border-t border-gray-700 p-4 font-mono text-sm overflow-y-auto">
      {/* 🎯 用 map 循环渲染日志，并根据日志的 type 改变字体颜色 */}
      {terminalLogs.map(log => (
        <p
          key={log.id}
          className={`mb-1 ${log.type === 'error' ? 'text-red-400' :
            log.type === 'success' ? 'text-green-400' :
              'text-gray-500'
            }`}
        >
          {log.text}
        </p>
      ))}
    </div>
  );
};

export default Terminal;
```
