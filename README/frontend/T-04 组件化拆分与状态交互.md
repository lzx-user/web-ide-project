# T-04：状态驱动 UI——让 IDE 拥有“记忆”

## 1、开发思路

### **第一步：理解 React 的“大脑”逻辑**

在传统网页里，你要改个名字得去手动操作 DOM，但在 React 里，我们只管**改变数据（State）**，界面会自动跟着动。

### **第二步：实现“所见即所得”的同步**

我们要确保：

1. 用户在左边点哪个文件，顶部的名字就得立刻换。
2. 用户在中间写什么代码，总部得立刻记住，不能丢。

------

## 2、实操步骤（手敲代码）

### **2.1 总部部署：定义核心状态**

**核心知识点（敲黑板）：** 为什么要把状态写在 `App.jsx` 而不是各个小组件里？因为这叫 **“状态提升”**。如果数据在子组件里，别的组件就看不到了。

```react
// App.jsx
function App() {
  // 定义 activeFile，默认是 index.js
  const [activeFile, setActiveFile] = useState('index.js'); 
  
  // 定义 currentCode，存储编辑器的实时内容
  const [currentCode, setCurrentCode] = useState('console.log("Hello!");'); 

  return (
    <div className="...">
      {/* 传给侧边栏：让它能修改总部的文件名 */}
      <Sidebar activeFile={activeFile} setActiveFile={setActiveFile} />
      
      {/* 传给顶部：让它显示当前文件名 */}
      <Header activeFile={activeFile} ... />
    </div>
  );
}
```

### **2.2 子组件响应：点击切换**

**为什么要写 `onClick`？** 因为这是用户发出的一个“信号”。 **为什么要调用 `setActiveFile`？** 因为子组件不能直接改总部的变量，必须通过这个总部给的“对讲机”申请修改。

```react
// Sidebar.jsx
<p
  onClick={() => setActiveFile(file.name)}
  className={`... ${activeFile === file.name ? 'bg-blue-600' : ''}`}
>
  {file.name}
</p>
```

2. 如果是，说明你已经掌握了 **React 数据流** 的精髓！
