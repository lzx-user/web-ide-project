WebSocket 是一种在单个 TCP 连接上进行**全双工（Full-Duplex）通信**的网络协议。它的出现主要是为了解决传统 HTTP 协议在实时通信场景下的痛点（例如：HTTP 只能由客户端发起请求，且每次请求都会携带较重的头部数据）。

以下是 WebSocket 协议基础的几个核心内容：

### 1. 握手阶段：借道 HTTP 实现协议升级

WebSocket 连接的建立非常巧妙，它最初是通过标准的 HTTP 请求发起的。客户端会在 HTTP 头部中明确告诉服务端：“我想升级到 WebSocket 协议”。

- **客户端请求：** 携带 `Connection: Upgrade` 和 `Upgrade: websocket` 头部字段，并附带一个随机生成的 `Sec-WebSocket-Key` 用于安全校验。
- **服务端响应：** 如果服务端支持 WebSocket，会返回 **HTTP 101 Switching Protocols** 状态码，并根据客户端发来的 Key 计算出 `Sec-WebSocket-Accept` 返回给客户端。
- **结果：** 握手成功后，HTTP 协议正式“退场”，底层的 TCP 连接被保留下来，后续的数据传输全部采用 WebSocket 的数据帧格式。

### 2. 全双工与持久连接（Full-Duplex & Persistent）

这是 WebSocket 最核心的价值所在。

- **持久性：** 一旦握手成功，连接就会一直保持打开状态，直到客户端或服务端主动关闭。
- **双向对等通信：** 服务端不再需要被动等待客户端的请求，它可以随时主动向客户端推送数据；客户端也可以随时发送数据。两者地位完全对等。

### 3. 数据帧传输（Data Framing）

为了减少网络开销，WebSocket 将消息拆分为一个或多个**帧（Frames）**进行发送。相比于 HTTP 每次都要携带几百字节的头部，WebSocket 的数据帧头部非常轻量（最小只有 2 字节）。

- **支持多种数据类型：** 它可以传输纯文本数据（通常是 UTF-8 编码的字符串，如 JSON），也可以传输二进制数据（如图片、音频流的 ArrayBuffer 或 Blob）。

### 4. 协议标识符（ws 与 wss）

就像 HTTP 和 HTTPS 一样，WebSocket 也有自己的统一资源标志符（URI）方案：

- **ws://**：非加密的 WebSocket 连接（默认端口 80）。
- **wss://**：加密的 WebSocket 连接（运行在 TLS/SSL 之上，默认端口 443），在生产环境中为了数据安全，通常强制要求使用 wss。

### 5. 心跳机制（Ping / Pong）

在长连接中，网络波动或路由器超时随时可能导致连接默默断开（俗称“死链接”）。WebSocket 协议在底层定义了专门的控制帧来维持连接：

- 一端可以发送 **Ping 帧**，另一端收到后必须立即回复 **Pong 帧**。
- 这种机制不仅能防止连接被网络设备强制回收，还能帮助双方实时检测对方是否“在线”。

### 📊 快速对比：WebSocket vs HTTP

| **特性**       | **传统 HTTP (1.0/1.1)**                            | **WebSocket**                             |
| -------------- | -------------------------------------------------- | ----------------------------------------- |
| **通信模式**   | 单工/半双工（请求-响应模式）                       | 全双工（双向实时通信）                    |
| **连接状态**   | 无状态，通常短连接（即使有 Keep-Alive 也是单向的） | 有状态，持久长连接                        |
| **服务端推送** | 不支持（需依赖轮询或长轮询）                       | **原生支持**                              |
| **数据开销**   | 每次请求都包含完整的 HTTP 头部，开销大             | 仅握手时有 HTTP 头部，后续数据帧头部极小  |
| **适用场景**   | 网页浏览、RESTful API 交互                         | 实时聊天、多人游戏、股票/加密货币行情推送 |
