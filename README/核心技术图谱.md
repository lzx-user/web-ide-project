### 🧱 Web IDE 核心技术图谱摘要

1. **前端现代工程化集群**
   - **React 19 & Zustand**：负责构建 IDE 的壳子（布局、状态调度）。Zustand 在这里充当了“中央指挥部”，全权管理房间号、Socket 实例、文件树和终端日志的存取。
   - **Monaco Editor (@monaco-editor/react)**：VS Code 的底层核心。它不是一个普通的 `<textarea>` 输入框，而是一个拥有独立 DOM 体系、语法高亮、代码解析、以及高频变动事件监听（`Model` 模型）的工业级编辑器。
   - **Allotment**：负责实现类似 VS Code 的窗口可拖拽自由分割布局。
2. **分布式数据协同集群（核心金玉）**
   - **Yjs & CRDT 算法**：解决“多人在同一行打字冲突”的分布式数学模型。它将文本转为单向链表，给每个字符发身份证。
   - **y-monaco & MonacoBinding**：胶水层。负责把 Monaco 用户的每一次键盘敲击（Delta 变动）实时翻译给 Yjs，同时把远程接收到的 Yjs 链表变动无损注入到 Monaco。
   - **y-indexeddb**：浏览器本地数据库。负责在用户的硬盘里默默记下每一次协同的日志，断线重连时提供历史轨迹。
3. **全双工网络通信集群（管道）**
   - **Socket.io**：控制面通道。负责瞬时状态的通知（如：大厅登录、创建文件、删除文件、点击运行代码、实时接收黑盒执行的日志流）。
   - **Native WebSocket**：数据面通道。专给 Yjs 跑高速公路，用来传输最纯粹的二进制协作数据。
   - **HTTP Upgrade（协议分流）**：后端的核心亮点。只开一个 3000 端口，通过嗅探 HTTP 请求头中的升级标志（Upgrade），优雅地将 Socket.io 流量和 Yjs WebSocket 流量分流到不同的处理器。
4. **后端沙箱与操作系统执行集群（硬核硬实力）**
   - **node-pty (Pseudo-Terminal)**：伪终端。在后端真正 fork（分叉）出系统底层的 PowerShell 或 Bash 进程，让前端的输入真正能在服务器上像操作黑盒子一样执行命令。
   - **Xterm.js**：前端终端渲染器。负责把后端 `node-pty` 吐出来的 ANSI 转义序列（带有各种颜色和控制符的原始文本流）完美渲染成黑底白字或白底黑字的极客终端面板。
   - **Child_process (exec)**：子进程执行流。独立开辟一个 5 秒超时的安全屋，专门用来跑用户的 `stream_*.js` 代码，与主 Node.js 服务完全隔离，防止死循环搞崩服务器。
