# Web IDE 开发笔记：多文件状态与终端隔离

> **项目背景**：在基于 React + Monaco Editor 开发类似 VS Code 的 Web IDE 时，实现多 Tab 页的文件切换，并保持各自的代码、撤销记录、光标位置和终端日志互不干扰。

## 一、 核心功能与实现思路

### 1. Monaco Editor 多文件状态隔离（防代码串台）
*   **痛点**：如果只用 `editor.setValue()` 替换文本，会导致 A 文件的撤销记录（Undo/Redo）和光标位置带到 B 文件中。
*   **核心思路（UI与数据分离）**：
    *   **编辑器（Editor）**：挂载在 DOM 上的外壳，全局只需实例化一次。
    *   **模型（Model）**：底层数据模型。每个文件对应一个独立的 Model（包含文本、语法、撤销栈）。
    *   **视图状态（ViewState）**：保存光标位置和滚动条进度。
*   **执行流程**：
    1.  **切出前**：调用 `editor.saveViewState()` 将当前文件的光标状态存入缓存柜。
    2.  **准备新文件**：去缓存柜找，没有就用 `monaco.editor.createModel` 创建新 Model。
    3.  **切入新文件**：调用 `editor.setModel()` 切换底层数据，并用 `editor.restoreViewState()` 恢复之前保存的光标位置。

### 2. 终端日志多文件隔离（防输出串门）
*   **痛点**：所有文件运行的结果都 push 到同一个大数组中，导致看 A 文件时能看到 B 文件的运行日志。
*   **核心思路（数据结构升维）**：
    *   将单一的数组 `[ {text: '...'} ]` 改为 **普通对象（字典树）**：
        `{ 'index.js': [{text: '...'}], 'package.json': [{text: '...'}] }`
    *   每次追加日志时，根据当前的 `activeFile` 作为键名（Key），将日志精准推入对应的数组中。

## 二、 核心知识点梳理

### 1. React 层面
*   **`useRef` 的三大神级用法**：
    1.  **绑定 DOM / 存实例**：存放 Monaco Editor 的底层实例，脱离 React 的渲染周期。
    2.  **做静默缓存（文件柜）**：存放 `fileCacheMap`。数据经常变，但绝对不能引发页面重绘（Re-render）。
    3.  **打破闭包陷阱（实时追踪器）**：用 `useRef` 包装 `activeFile`，让异步回调函数（如 Socket 监听）永远能读到最新的当前文件名，而不是被“定格”在组件挂载时的旧变量。
*   **`useState` 的函数式更新 (`prev`)**：
    *   在连续、高频更新状态时（如追加日志），严禁直接使用外部变量。必须使用 `setState(prev => ...)` 向 React 索要底层的最新状态，防止数据丢失覆盖。

### 2. Monaco Editor API 层面
*   `monaco.editor.create`：初始化编辑器外壳。
*   `monaco.editor.createModel(value, language, uri)`：创建独立文件模型，`uri` 极其重要，是跨文件语法提示的身份证。
*   `editor.setModel(model)`：执行模型替换的核心动作。
*   `model.dispose()`：销毁模型。在关闭文件 tab 或组件卸载时必须调用，否则严重内存泄漏。

```react
// App.jsx 完整核心代码 (已修复多文件串台、闭包陷阱、白屏报错)
import React, { useRef, useState, useEffect, useCallback } from 'react';
import { debounce } from 'lodash';
import axios from 'axios';

import Sidebar from './components/Sidebar';
import Header from './components/Header';
import CodeEditor from './components/CodeEditor';
import Terminal from './components/Terminal';
import Login from './components/Login';
import { socket, connectSocket } from './services/socket';
import request from './services/request';
import { Editor } from '@monaco-editor/react';

function App() {
  // ==========================================
  // 1. 核心 Refs 准备区 (文件柜与追踪器)
  // ==========================================
  const editorRef = useRef(null);  // 存放 Monaco 编辑器实例
  const monacoRef = useRef(null);  // 存放 Monaco 核心对象
  
  // 【绝密文件柜】：缓存每个文件的底层 Model 和 光标状态
  // 格式: { 'index.js': { model: 对象, viewState: 对象 } }
  const fileCacheMap = useRef(new Map()); 
  
  // 记录“上一次”看的是哪个文件（用来给旧文件存书签）
  const prevFileRef = useRef('index.js'); 

  // ==========================================
  // 2. React 状态区 (UI 驱动数据)
  // ==========================================
  const [activeFile, setActiveFile] = useState('index.js');
  
  // 【闭包克星：实时追踪器】：永远记录当前最新的 activeFile
  const activeFileRef = useRef(activeFile);
  useEffect(() => {
    activeFileRef.current = activeFile;
  }, [activeFile]);

  const [currentCode, setCurrentCode] = useState('');
  const [isSaving, setIsSaving] = useState(false);
  const [isRunning, setIsRunning] = useState(false);

  // 【终端隔离】：使用“普通对象”管理，文件名为 Key，日志数组为 Value
  const [terminalLogsMap, setTerminalLogsMap] = useState({
    'index.js': [{ id: Date.now(), type: 'info', text: '# 终端已就绪。点击顶部运行按钮执行代码。' }]
  });

  const [isJoined, setJoined] = useState(false);
  const [currentSocket, setCurrentSocket] = useState(null);
  const [initialRoomId, setInitialRoomId] = useState('');

  // ==========================================
  // 3. 核心业务与辅助函数
  // ==========================================

  // 向终端追加日志 (完美避开闭包陷阱)
  const addLog = (type, text) => {
    setTerminalLogsMap(prev => {
      // 🌟 永远从追踪器里拿最新文件名，绝不串门
      const targetFile = activeFileRef.current; 
      const currentLogs = prev[targetFile] || []; 
      
      // 使用对象解构语法返回新对象
      return {
        ...prev,
        [targetFile]: [...currentLogs, { id: Date.now(), type, text }]
      };
    });
  };

  // 秘书上岗：初始化捕获编辑器
  const handleEditorDidMount = (editor, monaco) => {
    editorRef.current = editor;
    monacoRef.current = monaco;

    // 创建第一张“纸”并放入柜子
    const initialModel = monaco.editor.createModel(
      currentCode || '', 
      'javascript',
      monaco.Uri.parse(`file://index.js`) 
    );
    editor.setModel(initialModel);

    fileCacheMap.current.set('index.js', {
      model: initialModel,
      viewState: null
    });
  };

  // 登录/加入房间
  const handleJoinRoom = async (usernameInput, roomIdInput) => {
    try {
      const { data } = await request.post('/join', { username: usernameInput, roomId: roomIdInput });
      if (data.success) {
        localStorage.setItem('ide_token', data.token);
        window.history.pushState({}, '', `?roomId=${roomIdInput}`);
        const s = connectSocket(roomIdInput, data.token);
        setCurrentSocket(s);
        setJoined(true);
      } else {
        alert(data.message);
      }
    } catch (err) {
      addLog('error', '系统错误: 无法建立连接');
    }
  };

  // 保存代码
  const handleSave = async () => {
    if (!editorRef.current || isSaving) return;
    setIsSaving(true);
    addLog('info', '正在保存代码');
    try {
      const code = editorRef.current.getValue();
      await request.post('/save', { roomId: initialRoomId || 'A2026', code, language: 'javascript' });
      addLog('info', '保存成功');
    } catch (err) {
      addLog('error', '保存失败: ' + err.message);
    } finally {
      setIsSaving(false);
    }
  };

  // 运行代码
  const handleRun = async () => {
    if (!editorRef.current || isRunning) return;
    setIsRunning(true);
    const code = editorRef.current.getValue();
    addLog('info', '正在运行代码...');
    if (currentSocket) currentSocket.emit('executeCode', code);
    try {
      await request.post('/run', { roomId: initialRoomId || 'A2026', code, language: 'javascript' });
    } catch (err) {
      addLog('error', '运行失败: ' + err.message);
    } finally {
      setIsRunning(false);
    }
  };

  // 代码变更防抖
  const debouncedEmitCode = useCallback(
    debounce((code) => {
      if (currentSocket) currentSocket.emit('codeChange', code);
    }, 500),
    [currentSocket]
  );

  const handleCodeChange = (newCode) => {
    setCurrentCode(newCode);
    debouncedEmitCode(newCode);
  };

  // ==========================================
  // 4. 副作用监听区 (useEffect)
  // ==========================================

  // URL 房间初始化
  useEffect(() => {
    const urlParams = new URLSearchParams(window.location.search);
    const roomFromUrl = urlParams.get('roomId');
    if (roomFromUrl) setInitialRoomId(roomFromUrl);
  }, []);

  // Socket 消息监听
  useEffect(() => {
    if (!currentSocket) return;

    const handleOutput = (data) => addLog('info', data);
    const handleError = (data) => addLog('error', data);
    const handleFinish = (exitCode) => addLog('info', `\n[进程执行完毕，退出码: ${exitCode}]`);

    currentSocket.on('terminalOutput', handleOutput);
    currentSocket.on('terminalError', handleError);
    currentSocket.on('executionFinished', handleFinish);

    return () => {
      currentSocket.off('terminalOutput', handleOutput);
      currentSocket.off('terminalError', handleError);
      currentSocket.off('executionFinished', handleFinish);
    };
  }, [currentSocket]);

  // 【核心机制：多文件切换与隔离】
  useEffect(() => {
    const editor = editorRef.current;
    const monaco = monacoRef.current;
    if (!editor || !monaco) return;

    const prevFile = prevFileRef.current;
    const targetFile = activeFile;

    // 1. 保存旧文件的现场 (光标书签)
    if (prevFile && fileCacheMap.current.has(prevFile)) {
      const currentState = editor.saveViewState();
      const oldCache = fileCacheMap.current.get(prevFile);
      oldCache.viewState = currentState; 
    }

    // 2. 去柜子里找新文件
    let targetCache = fileCacheMap.current.get(targetFile);

    // 3. 如果是第一次打开，新建底层模型 (Model)
    if (!targetCache) {
      const newModel = monaco.editor.createModel(
        '', 
        'javascript',
        monaco.Uri.parse(`file://${targetFile}`) 
      );
      targetCache = { model: newModel, viewState: null };
      fileCacheMap.current.set(targetFile, targetCache);
    }

    // 4. 铺上新纸，恢复书签
    editor.setModel(targetCache.model);
    if (targetCache.viewState) {
      editor.restoreViewState(targetCache.viewState);
    }

    // 5. 更新指针，等待下次切换
    prevFileRef.current = targetFile;

  }, [activeFile]);

  // ==========================================
  // 5. 渲染区 (UI)
  // ==========================================

  if (!isJoined) {
    return <Login onJoinRoom={handleJoinRoom} initialRoomId={initialRoomId} />;
  }

  return (
    <div className="h-screen w-screen flex bg-gray-900 text-white font-sans">
      <Sidebar activeFile={activeFile} setActiveFile={setActiveFile} />

      <div className="flex-1 flex flex-col">
        <Header
          activeFile={activeFile}
          isSaving={isSaving}
          isRunning={isRunning}
          onSave={handleSave}
          onRun={handleRun}
        />

        <CodeEditor
          code={currentCode}
          setCode={handleCodeChange}
          roomId={initialRoomId}
          socket={currentSocket}
          onMount={handleEditorDidMount}
        />
        
        {/* 只渲染当前激活文件的专属日志 */}
        <Terminal logs={terminalLogsMap[activeFile] || []} />
      </div>
    </div>
  );
}

export default App;
```

## 三、 遇到的问题与破局（Debug 记录）

### 🐛 Bug 1：页面纯白屏 (White Screen of Death)
*   **报错现象**：`Uncaught ReferenceError: Cannot access 'activeFile' before initialization`
*   **原因分析**：JavaScript 从上往下执行。在 `useState` 创建 `activeFile` 之前，就试图把它传给 `useRef` 导致直接崩溃；此外，JSX 的 `return` 中不小心混入了纯中文文本。
*   **解决方案**：调整代码顺序，确保变量先声明后使用；清理 JSX 中的非法语素。

### 🐛 Bug 2：追加日志报错 `prev.get is not a function`
*   **原因分析**：数据类型张冠李戴。初始化时用的是普通数组 `[]`，但在更新逻辑中却把它当成了 ES6 的 `Map` 对象调用了 `.get()` 方法。
*   **解决方案**：统一使用**普通对象 `{}`** 来管理字典映射。利用对象解构语法更新数据：`{ ...prev, [fileName]: [...oldLogs, newLog] }`。

### 🐛 Bug 3：终极 Boss —— 异步闭包陷阱 (Stale Closure)
*   **报错现象**：代码不报错，但在 `package.json` 触发运行后，执行结果却打印在了 `index.js` 的终端界面上。
*   **原因分析**：监听 WebSocket 回调的 `useEffect` 仅在组件初始化时运行了一次。此时它绑定了初始状态 `activeFile = 'index.js'`。由于闭包特性，此后无论 UI 如何切换文件，Socket 收到消息时始终往 `index.js` 里塞日志。
*   **解决方案**：
    ```javascript
    // 建立实时追踪器
    const activeFileRef = useRef(activeFile);
    useEffect(() => { activeFileRef.current = activeFile; }, [activeFile]);
    
    // 在 Socket 回调中读取
    const targetFile = activeFileRef.current; // 永远获取最新值
    ```
