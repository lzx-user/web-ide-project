# T-05：打通前后端 API——发送代码并运行

## 1、开发思路

### **第一步：后端开启“跑腿服务”**

后端 Node.js 需要准备一个接口（API），专门接收前端发来的代码，并在服务器上跑一下，最后把结果送回来。

### **第二步：前端“下单”并“等餐”**

前端通过 `fetch` 向后端发起请求。在这个过程中，我们要处理三种情况：成功了、报错了、服务器挂了。

------

## 2、实操步骤（手敲代码）

### **2.1 总部编写：handleRunCode 函数**

**核心知识点（敲黑板）：** * **`async/await`**：因为网络请求是需要时间的，我们要“异步”等待结果。

- **`try...catch...finally`**：这是外企最看重的**健壮代码**习惯。无论成功失败，最后都要把按钮的 `isRunning` 设回 `false`。

```react
// App.jsx
const handleRunCode = async () => {
  if (!currentCode.trim()) return; // 没写代码就不发请求

  setIsRunning(true); // 按钮变灰，开始转圈
  addLog('info', `> 正在执行 ${activeFile} ...`); // 终端提示

  try {
    // 1. 发起请求
    const response = await fetch('http://localhost:3000/api/run', {
      method: 'POST', // 用 POST，因为代码内容可能很长
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ code: currentCode }) // 把代码塞进信封
    });

    const result = await response.json();

    // 2. 处理结果：后端说成功就变绿，失败就变红
    if (result.success) {
      addLog('success', result.output);
    } else {
      addLog('error', result.output);
    }
  } catch (err) {
    // 3. 处理意外：比如后端没开
    addLog('error', '❌ 连接服务器失败！');
  } finally {
    setIsRunning(false); // 无论如何，最后都要释放按钮
  }
};
```

## 3、验证成果

1. 在编辑器写一句 `console.log("123")`。
2. 点击运行按钮，看按钮是否变灰显示“⏳ 运行中...”。
3. 看底部终端是否出现了绿色的 `123`。
